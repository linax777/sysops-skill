### 🚨 1. 檔案過大警告 (Large Files >500 行)
*超過 500 行的檔案會導致 Agent 的 Context Window 壓力增加，容易產生幻覺或遺漏片段。*
- `[檔案路徑A]` (原因：XXX)
- `[檔案路徑B]` (原因：XXX)
👉 **改善建議**：考慮將此檔案依邏輯拆分為多個 `.md`，並在當前目錄的 `index.md` 中增加這幾份新檔案的摘要引導。

### 🏷️ 2. 缺少標籤 (Missing Tags in Leaf `index.md`)
*底層目錄的 index.md 必須使用 `Tags: [...]` 來標記每個檔案資源，這能讓 Agent 直接判定目標而不需猜測。*
- `[檔案路徑A]`
  - 第 X 行缺乏 Tags 標籤: [該行內容片段]
👉 **改善建議**：請在每一行資源描述中加入 `Tags: [關鍵字1, 關鍵字2]`。

### 📄 3. 不良的檔案格式 (Invalid File Formats)
*知識庫中不應該混入 PDF, Word 或 Excel 等二進位檔案，這會導致檢索失敗。*
- `[檔案路徑A]` 
👉 **改善建議**：請轉為純文字格式，系統參數建議轉為 JSON/YAML，文件請轉為 Markdown。
